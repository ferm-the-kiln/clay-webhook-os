  <action>
Create the batch results page at `dashboard/src/app/batch-results/page.tsx`.

Follow the exact pattern from `/run/page.tsx`: "use client", Suspense wrapper around inner component, Header from `@/components/layout/header`.

The page must:
1. Read `id` from URL search params via `useSearchParams()` from `next/navigation`
2. Show a loading state while fetching
3. If no `id` param, show a message: "No batch ID provided. Navigate here from the Run page after completing a batch."
4. On mount with a valid `id`, call `fetchBatchStatus(id)` to get the batch summary
5. If `batchStatus.done` is true, fetch full job details: `Promise.all(batchStatus.jobs.map(j => fetchJob(j.id)))` and store in state
6. If `batchStatus.done` is false, poll every 2 seconds using `setInterval` + `useEffect` cleanup, fetching batch status until done, then fetch all full jobs
7. Extract CSV headers from the first completed job's input data (the `data` field sent to the webhook) -- since batch results page has no original CSV, pass empty arrays for `originalRows` and `csvHeaders` (the SpreadsheetView handles this gracefully, showing only output columns)
8. Render `<SpreadsheetView jobs={jobs} originalRows={[]} csvHeaders={[]} />` once jobs are loaded
9. Above the spreadsheet, show a summary bar: batch_id (truncated), total rows, completed count, failed count, and cost (from `batchStatus.cost.equivalent_api_usd`)
