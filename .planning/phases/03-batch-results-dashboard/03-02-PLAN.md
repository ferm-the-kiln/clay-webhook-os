**A. Add confidence-based row coloring to `spreadsheet-row.tsx`:**

Add a `getConfidenceColor` helper function (can be defined above the component or in the same file):
```typescript
function getConfidenceColor(score: number | undefined): string {
  if (score === undefined || score === null) return "";
  if (score >= 0.7) return "bg-status-success/5 hover:bg-status-success/10";
  if (score >= 0.4) return "bg-kiln-mustard/5 hover:bg-kiln-mustard/10";
  return "bg-kiln-coral/5 hover:bg-kiln-coral/10";
}
```

In `SpreadsheetRowComponent`, extract confidence from the job result:
```typescript
const confidence = typeof job.result?.confidence_score === "number"
  ? job.result.confidence_score
  : typeof job.result?.overall_confidence_score === "number"
    ? job.result.overall_confidence_score
    : undefined;
const confidenceClass = getConfidenceColor(confidence);
```

Apply `confidenceClass` to the `<tr>` className. The confidence color should be the BASE background (lowest priority), with selection (kiln-teal) and expansion (clay-800) taking precedence. Update the className logic:
```typescript
className={`border-b border-clay-500 cursor-pointer transition-colors ${
  isSelected
    ? "bg-kiln-teal/5 hover:bg-kiln-teal/10"
    : confidenceClass || "hover:bg-clay-800/50"
} ${expanded ? "bg-clay-800/30" : ""}`}
```

Also add an `onRowClick` optional prop to `SpreadsheetRowComponent`. When provided, clicking the row calls `onRowClick(job)` instead of toggling expansion. When NOT provided (backward-compatible), clicking still toggles the inline RowDetailPanel as before.
