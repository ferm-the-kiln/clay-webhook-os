## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Gap between icon and label in message bubbles, inline badge padding |
| sm | 8px | Gap between messages in thread, inner padding of input bar, compact element spacing |
| md | 16px | Page padding on mobile (`p-4`), message bubble horizontal padding, section gaps |
| lg | 24px | Page padding on desktop (`p-6`), gap between chat thread and activity panel |
| xl | 32px | Message bubble vertical group spacing (between user/assistant pairs) |
| 2xl | 48px | Empty state vertical padding |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Session list panel width: 280px (w-70) collapsed, toggleable
- Activity panel placeholder width: 320px (w-80) on lg+ screens, hidden below lg
- Chat input textarea min-height: 44px (touch target), max-height: 160px (4 lines)

## Layout Contract

### Page structure (`/chat`)

```
+------------------------------------------------------------------+
| Header: "Chat"                                                    |
+----------+---------------------------+---------------------------+
| Session  |  Chat Thread              |  Activity Panel           |
| List     |                           |  (placeholder)            |
| (280px)  |  +---------------------+  |                           |
| toggle-  |  | FunctionPicker      |  |  EmptyState:              |
| able     |  +---------------------+  |  "Activity will appear    |
|          |  | Messages...         |  |   here during execution"  |
| [New     |  | User bubble         |  |                           |
|  Chat]   |  | Assistant bubble    |  |                           |
|          |  |   + OutputView      |  |                           |
| Session1 |  |                     |  |                           |
| Session2 |  +---------------------+  |                           |
| Session3 |  | ChatInput           |  |                           |
|          |  | [textarea] [Send]   |  |                           |
+----------+---------------------------+---------------------------+
```

### Responsive behavior:
- **lg+ (1024px+):** Three-column layout. Session list (280px, collapsible) + chat thread (flex-1) + activity panel (320px).
- **md (768-1023px):** Two-column. Session list hidden (toggle via button in header area). Chat thread (flex-1) + activity panel hidden.
- **Below md:** Single column. Chat thread only. Session list accessible via sheet/drawer. Activity panel hidden.

### CSS approach:
```
// Outer container
flex flex-col h-full

// Content area below header
flex-1 overflow-hidden flex

// Session list (collapsible)
w-70 border-r border-clay-600 bg-clay-800 flex-col
hidden when collapsed, shown via state toggle

// Chat thread (main)
flex-1 flex flex-col min-w-0

// Activity panel placeholder
hidden lg:block w-80 border-l border-clay-600
```

## Interaction Contract

### Keyboard shortcuts:
| Shortcut | Action |
|----------|--------|
| Cmd+2 | Navigate to /chat (new sidebar shortcut) |
| Enter | Send message (when input focused and has content) |
| Shift+Enter | Insert newline in input |
| Escape | Close function picker popover |

### Auto-scroll behavior:
- Chat thread auto-scrolls to bottom when a new message is added (user or assistant).
- If user has scrolled up (more than 100px from bottom), auto-scroll is suppressed.
- A "scroll to bottom" button appears when user is scrolled up and new messages arrive.
- Scroll-to-bottom button: small pill at bottom-center of thread, `bg-clay-700 text-clay-200`, shows down-arrow icon.

### Streaming UX:
1. User sends message: user bubble appears immediately (optimistic).
2. Assistant placeholder bubble appears with "Processing..." text and a pulsing kiln-teal dot.
3. SSE events update the placeholder: "Processing 1/12...", "Processing 2/12..."
4. On `function_complete`: placeholder replaced with final assistant message containing structured results.
5. On `error`: placeholder shows error copy in `text-kiln-coral` with retry guidance.
6. AbortController stored in `useRef`, cleaned up on unmount and session switch.

### Session switching:
1. Click session in list: current stream (if active) is aborted via AbortController.
2. Session messages load from `GET /channels/{id}`.
3. Thread scrolls to bottom showing loaded history.
4. Function picker updates to show the session's function (read-only display, not changeable mid-session).

### Data parsing (v1 -- simple):
- Input text split by newlines.
- Each non-empty line becomes `{"value": "<line>"}` in the data array.
- No CSV parsing in v1 -- that is the Workbench's job.
- The key name `"value"` is used universally; the backend/function handles mapping.
